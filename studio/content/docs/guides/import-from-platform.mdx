AgentFlow can import configurations from six supported platforms, converting their rules, instructions, and project context into the `.agentflow/` directory structure.

## Import Workflow

<Steps>
<Step>
### Auto-detect platform

Run the import command from your project root. AgentFlow will detect which platform configuration is present:

```bash
agentflow import --source ./my-project
```

If multiple platforms are detected, the one with the highest confidence score is selected. You can also specify the platform explicitly:

```bash
agentflow import --platform cursor --source ./my-project
```
</Step>

<Step>
### Run the import

The import pipeline executes in order:

1. Validate source directory
2. Apply reverse transforms (platform format to AgentFlow format)
3. Validate output paths
4. Check for existing `AGENTS.md`

</Step>

<Step>
### Review output

Inspect the generated `.agentflow/` directory to confirm the imported structure matches your expectations.

<Files>
  <Folder name=".agentflow" defaultOpen>
    <File name="AGENTS.md" />
    <Folder name="instructions">
      <File name="imported-rules.md" />
    </Folder>
    <Folder name="memory">
      <File name="context.md" />
    </Folder>
  </Folder>
</Files>

</Step>

<Step>
### Validate

Run validation to ensure the imported workflow is well-formed:

```bash
agentflow validate
```
</Step>
</Steps>

## Auto-detection

When no `--platform` flag is provided, AgentFlow runs `detectPlatform()` which scans the source directory for marker files. Each platform has known markers:

| Platform | Marker Files |
|----------|-------------|
| Kiro | `.kiro/steering` |
| Cursor | `.cursor/rules`, `.cursorrules` |
| Claude Code | `CLAUDE.md`, `.claude/` |
| VS Code Copilot | `.github/copilot-instructions.md` |
| Windsurf | `.windsurf/rules`, `.windsurfrules` |
| OpenClaw | `SOUL.md`, `HEARTBEAT.md` |

Each detected marker contributes to a confidence score. The platform with the highest score is selected for import.

## Dry-run Mode

Use `--dry-run` to preview what the import would produce without writing any files:

```bash
agentflow import --source ./my-project --dry-run
```

This outputs the list of files that would be created and any warnings, without modifying your filesystem.

<Callout type="warn">
Automated imports may not capture every nuance of your platform configuration. After importing, review the generated files and make manual adjustments — particularly for complex conditional rules, platform-specific tool integrations, or context that relies on runtime state.
</Callout>
