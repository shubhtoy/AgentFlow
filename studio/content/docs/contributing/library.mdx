The library contains pre-built workflows and resources that users can install into their workspace. Contributions here help the entire community.

<Callout type="info">
💡 Library content is auto-indexed. Run `node scripts/build-library-index.js` or it runs automatically via prebuild. No manual manifest files needed.
</Callout>

The five resource categories are: **instructions**, **capabilities**, **skills**, **memory**, and **hooks**.

Skills are directory-based — each skill is a directory containing its files (e.g., `SKILL.md`, resources). Skills are imported from [skills.sh](https://skills.sh).

## Auto-Indexing

The library is auto-indexed by `scripts/build-library-index.js`, which generates `library/index.json`. This script:

- Runs automatically as a prebuild step (`predev` and `prebuild` in `studio/package.json`)
- Walks the entire `library/` directory recursively
- Produces a flat list of all file paths
- Eliminates the need for per-directory manifest files

The studio serves library content via `studio/public/library`, which is a symlink to `../../library`. Changes to library files are immediately reflected in the studio without rebuilding.

## Naming Conventions

- Use lowercase kebab-case for all directory and file names
- Workflow directories should describe the action: `build-feature`, `review-pr`, `debug-issue`
- Resource files should describe their purpose: `code-review.md`, `deploy-checklist.md`
- Node directories within workflows should be verb-first: `gather-requirements`, `validate-output`
- Skill directories should be descriptive: `systematic-debugging`, `code-review`, `frontend-design`

## Adding a Resource

<Steps>
  <Step>
    ### Create the resource

    Add your workflow or resource file to the appropriate subdirectory under `library/`. Follow the markdown format used by existing entries. For skills, create a directory with at least a `SKILL.md` file.
  </Step>
  <Step>
    ### Rebuild the index

    Run the library indexer to verify your addition is discovered correctly:

    ```bash
    node scripts/build-library-index.js
    ```

    Confirm your resource appears in `library/index.json`.
  </Step>
  <Step>
    ### Submit a PR

    Open a pull request with your addition. Include a brief description of what the resource does and when someone would use it.
  </Step>
</Steps>
