## Prerequisites

- Node.js 20 or later
- npm 9 or later
- Git

## Setup

<Steps>
  <Step>
    ### Clone the repository

    ```bash
    git clone https://github.com/agentflow/agentflow.git
    cd agentflow
    ```
  </Step>
  <Step>
    ### Install dependencies

    Install root dependencies first, then studio dependencies:

    ```bash
    npm install
    cd studio && npm install
    cd ..
    ```
  </Step>
  <Step>
    ### Run the dev server

    From the project root:

    ```bash
    npm run dev
    ```
  </Step>
  <Step>
    ### Run tests

    From the project root:

    ```bash
    npm test
    ```
  </Step>
  <Step>
    ### Run the studio

    To start the visual editor independently:

    ```bash
    cd studio && npm run dev
    ```

    Open `http://localhost:3000` in your browser.
  </Step>
</Steps>

## Project Structure Deep Dive

### `src/` - Core Engine

The core engine is a pure Node.js library with no framework dependencies. It handles parsing markdown into an AST, validating workflow structure, and exporting to target platforms. This directory has zero browser dependencies and can run in any Node.js environment.

- `parser.js` / `parser-core.js` - Reads `.md` files, extracts frontmatter, builds the workflow graph AST
- `validator.js` - Runs structural and semantic checks (broken refs, cycles, schema violations)
- `exporter.js` - Orchestrates the export pipeline from AST to platform-specific output
- `taxonomy.js` - Defines the fixed resource categories (instructions, capabilities, runbooks, memory, hooks)
- `library.js` - Handles installing and managing library templates
- `errors.js` - Centralized error codes and message formatting
- `pretty-printer.js` - Formats AST back to markdown for round-trip editing

### `src/services/` - Application Logic

Services wrap the core engine with higher-level operations. Each service is a plain object with async methods that return `ok`/`fail` results. Services are stateless and receive all dependencies through the context object.

### `src/transport/` - Platform Adapters

Contains JSON configuration files for each supported export target plus the generic adapter that interprets them. The pipeline files handle the full export/import orchestration including file I/O and path resolution.

### `src/git/` - Git Operations

Manages repository state, scanning for changes, configuration, and remote synchronization. Used by both the CLI and the studio for git-related features.

### `src/mcp/` - MCP Integration

Handles Model Context Protocol server lifecycle, tool discovery, and bridging external tools into the workflow context.

### `studio/` - Visual Editor

A Next.js 14 application using the App Router. Key subdirectories:

- `components/` - React components for the canvas, panels, dialogs, and editors
- `app/` - Next.js route handlers and page components
- `lib/` - Shared utilities, API clients, and type definitions
- `hooks/` - Custom React hooks for workflow operations and UI state
- `store/` - Zustand stores for workflow state, UI state, and settings

### `library/` - Reusable Templates

Pre-built content that users can install into their workspace. Organized by resource category. Each item is a standalone markdown file with complete frontmatter.

### `bin/` - CLI Entry Point

The `cli.js` file bootstraps the command-line interface. It parses arguments, loads configuration, creates the service layer, and dispatches to the appropriate command handler.

### `tests/` - Test Suite

Unit and integration tests organized to mirror the `src/` directory structure.

## Running Tests

The project uses Vitest as its test framework. Tests live in the `tests/` directory and follow the naming convention `*.test.js`.

### Run all tests

```bash
npm test
```

### Run tests in watch mode

```bash
npm run test:watch
```

### Run a specific test file

```bash
npx vitest run tests/parser.test.js
```

### Run tests with coverage

```bash
npx vitest run --coverage
```

Tests use in-memory file system mocks so they do not touch the real filesystem. Service tests use a mock context with stubbed dependencies.

## Common Development Tasks

### Adding a New CLI Command

1. Create a new handler function in `src/services/` or an existing service file
2. Register the command in `bin/cli.js` by adding a new case to the command dispatcher
3. Add argument parsing for any flags the command accepts
4. Return an `ok`/`fail` result from the handler
5. Add a test in `tests/` covering the success and failure paths

```js
// In bin/cli.js - register the command
case "my-command":
  return services.myService.run(args)
```

### Adding a New API Route

1. Create a new route file in `studio/app/api/` following the Next.js App Router convention
2. Import the service layer and call the appropriate service method
3. Return a JSON response using `NextResponse.json()`
4. Handle errors by checking the `ok` field on the service result

```js
// studio/app/api/my-route/route.js
import { createServiceLayer } from "@/lib/services"
import { NextResponse } from "next/server"

export async function POST(request) {
  const body = await request.json()
  const services = createServiceLayer(getContext())
  const result = await services.myService.run(body)

  if (!result.ok) {
    return NextResponse.json(result.error, { status: 400 })
  }
  return NextResponse.json(result.data)
}
```

### Adding a New Component

1. Create the component file in `studio/components/`
2. Use TypeScript props interface for type safety
3. Follow the existing pattern of co-locating styles with Tailwind classes
4. If the component needs workflow state, use the Zustand store via the appropriate hook
5. Export the component and use it in the relevant page or panel

```tsx
// studio/components/my-component.tsx
interface MyComponentProps {
  title: string
  onAction: () => void
}

export function MyComponent({ title, onAction }: MyComponentProps) {
  return (
    <div className="rounded-lg border p-4">
      <h3 className="text-sm font-medium">{title}</h3>
      <button onClick={onAction} className="mt-2 text-xs text-blue-600">
        Run
      </button>
    </div>
  )
}
```
