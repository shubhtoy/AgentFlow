## Setup

<Steps>
  <Step>
    ### Clone the repository

    ```bash
    git clone https://github.com/shubhtoy/agentflow.git
    cd agentflow
    ```
  </Step>
  <Step>
    ### Install dependencies

    From the project root, a single install handles all workspaces and the studio:

    ```bash
    npm install
    cd studio && npm install && cd ..
    ```
  </Step>
  <Step>
    ### Run the dev server

    From the project root:

    ```bash
    npm run dev
    ```

    This starts the studio at `http://localhost:3000`. The prebuild script (`scripts/build-library-index.js`) runs automatically before dev and build, generating `library/index.json`.
  </Step>
  <Step>
    ### Run tests

    ```bash
    npm test
    ```
  </Step>
</Steps>

<Callout type="info">
💡 `studio/public/library` is a symlink to `../../library`. Changes to `library/` are immediately reflected in the studio.
</Callout>

### `packages/core` — Core Engine

A pure TypeScript library with no framework dependencies. Handles parsing markdown into an AST, validating workflow structure, and defining the taxonomy. Zero browser dependencies — safe to run in the browser.

- `parser-core.ts` — Reads `.md` files, extracts frontmatter, builds the workflow graph AST. Nodes are `step` or `sub-workflow` types; router behavior is inferred
- `parser-browser.ts` — Thin browser-safe shim delegating to the core parser
- `validator.ts` — Structural and semantic checks (broken refs, cycles, schema violations)
- `taxonomy.ts` — Fixed resource categories: instructions, capabilities, skills, memory, hooks
- `errors.ts` — Centralized error codes and message formatting
- `services/` — Higher-level operations wrapping the core engine
- `mcp/` — MCP registry client

### `packages/cli` — CLI

Command-line interface for AgentFlow. Bootstraps commands, loads configuration, and dispatches to service handlers.

- `bin/cli.js` — Entry point, argument parsing, command dispatch
- `parser.ts` — CLI-specific parsing extensions
- `library.ts` — Installing and managing library content
- `pretty-printer.ts` — Formats AST back to markdown for round-trip editing
- `export/` — Export engine and transform pipeline. Reads YAML configs from `configs/platforms/`
- `services/` — Application logic for CLI commands
- `git/` — Repository state, change scanning, remote sync
- `mcp/` — MCP integration for CLI context

### `studio/` — Visual Editor

A Next.js 16 application using the App Router.

- `components/` — React components for canvas, panels, dialogs, editors
- `app/` — Route handlers and page components
- `lib/` — Shared utilities, API clients, type definitions
- `hooks/` — Custom React hooks for workflow operations and UI state
- `store/` — Zustand stores for workflow state, UI state, settings

## Running Tests

The project uses Vitest. Tests live in `tests/` and follow the `*.test.js` convention.

<Tabs items={["All tests", "Watch mode", "Single file", "Coverage"]}>
  <Tab value="All tests">
    ```bash
    npm test
    ```
  </Tab>
  <Tab value="Watch mode">
    ```bash
    npm run test:watch
    ```
  </Tab>
  <Tab value="Single file">
    ```bash
    npx vitest run tests/parser.test.js
    ```
  </Tab>
  <Tab value="Coverage">
    ```bash
    npx vitest run --coverage
    ```
  </Tab>
</Tabs>

Tests use in-memory file system mocks and do not touch the real filesystem.

## Common Development Tasks

### Adding a New CLI Command

1. Create a handler in `packages/cli/src/services/`
2. Register the command in `packages/cli/bin/cli.js`
3. Add argument parsing for any flags
4. Return an `ok`/`fail` result from the handler
5. Add a test in `tests/`

```ts
// In packages/cli/bin/cli.js
case "my-command":
  return services.myService.run(args)
```

### Adding a New API Route

1. Create a route file in `studio/app/api/` following the App Router convention
2. Import the service layer and call the appropriate method
3. Return JSON via `NextResponse.json()`

```ts
// studio/app/api/my-route/route.ts
import { createServiceLayer } from "@/lib/services"
import { NextResponse } from "next/server"

export async function POST(request: Request) {
  const body = await request.json()
  const services = createServiceLayer(getContext())
  const result = await services.myService.run(body)

  if (!result.ok) {
    return NextResponse.json(result.error, { status: 400 })
  }
  return NextResponse.json(result.data)
}
```

### Adding a New Component

1. Create the component in `studio/components/`
2. Use TypeScript props interface for type safety
3. Use Tailwind for styling, Zustand hooks for workflow state

```tsx
// studio/components/my-component.tsx
interface MyComponentProps {
  title: string
  onAction: () => void
}

export function MyComponent({ title, onAction }: MyComponentProps) {
  return (
    <div className="rounded-lg border p-4">
      <h3 className="text-sm font-medium">{title}</h3>
      <button onClick={onAction} className="mt-2 text-xs text-blue-600">
        Run
      </button>
    </div>
  )
}
```
