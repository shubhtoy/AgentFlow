## JSON Format

```json
{
  "name": "MyAgent",
  "dir": ".myagent",
  "cli": "myagent",
  "logo": "./assets/logo.svg",
  "theme": {
    "primary": "#6366f1",
    "font": "Inter"
  }
}
```

## Environment Variable Overrides

| Variable | Overrides | Example |
| --- | --- | --- |
| `AGENTFLOW_BRAND_NAME` | `name` | `AGENTFLOW_BRAND_NAME=MyAgent` |
| `AGENTFLOW_DIR` | `dir` | `AGENTFLOW_DIR=.myagent` |
| `AGENTFLOW_CLI` | `cli` | `AGENTFLOW_CLI=myagent` |

## Precedence

Resolution order from highest to lowest priority:

1. Environment variables
2. `agentflow.config.json`
3. Built-in defaults

## Defaults

| Field | Default |
| --- | --- |
| `name` | AgentFlow |
| `dir` | .agentflow |
| `cli` | agentflow |

<Callout type="warn">
If `agentflow.config.json` is invalid or unparseable, a warning is logged and built-in defaults are used. However, if an environment variable override contains an invalid value, the process exits with a fatal error.
</Callout>
