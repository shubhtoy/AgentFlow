title: Platform Configs

Each platform export is defined by a YAML configuration file in `configs/platforms/`. The config maps AgentFlow concepts to the native configuration format of the target platform. Adding a new platform requires only a new YAML config — no code changes.

<Tabs items={["agent-spec", "claude-code", "cursor", "kiro", "windsurf", "vscode-copilot", "openclaw"]}>
<Tab value="agent-spec">

## agent-spec

The primary interchange format. Exports the full workflow graph as an Agent Spec JSON file — recommended for cross-platform workflow sharing.

<Files>
  <File name="agent-spec.json" />
</Files>

**Key Transforms:**

- Complete workflow graph serialized to Agent Spec format
- All nodes, edges, conditions, and metadata included
- Resources inlined or referenced per spec requirements

**Fidelity:** High. Full graph representation without loss.

</Tab>
<Tab value="claude-code">

## claude-code

Exports to Claude Code's native `.claude/` directory structure.

<Files>
  <Folder name=".claude" defaultOpen>
    <Folder name="agents" defaultOpen>
      <File name="skill-name.md" />
    </Folder>
    <File name="settings.json" />
  </Folder>
  <File name="CLAUDE.md" />
  <File name=".mcp.json" />
</Files>

**Key Transforms:**

- Identity from `AGENTS.md` written to `CLAUDE.md` with identity markers stripped
- Skills exported as individual files in `.claude/agents/`
- MCP capabilities mapped to `.mcp.json`
- Hooks converted to `.claude/settings.json`

**Fidelity:** High. Most AgentFlow concepts have direct equivalents.

**Import Support:** Yes. Can import existing `.claude/` configurations into AgentFlow.

</Tab>

## cursor

Exports to Cursor's `.cursor/` directory structure.

<Files>
  <Folder name=".cursor" defaultOpen>
    <Folder name="rules" defaultOpen>
      <File name="skill-name.mdc" />
    </Folder>
    <File name="mcp.json" />
    <File name="hooks.json" />
  </Folder>
  <File name="AGENTS.md" />
</Files>

**Key Transforms:**

- Identity passed through as `AGENTS.md` (no transformation)
- Skills converted to MDC format in `.cursor/rules/`
- MCP capabilities mapped to `.cursor/mcp.json`
- Hooks converted to `.cursor/hooks.json`

**Fidelity:** High. MDC format supports metadata frontmatter for rule configuration.

**Import Support:** Yes. Can import existing `.cursor/` configurations into AgentFlow.

</Tab>

## vscode-copilot

Exports to VS Code Copilot's distributed configuration structure.

<Files>
  <Folder name=".github" defaultOpen>
    <Folder name="instructions" defaultOpen>
      <File name="skill-name.instructions.md" />
    </Folder>
    <Folder name="hooks" defaultOpen>
      <File name="agentflow-hooks.json" />
    </Folder>
    <File name="copilot-instructions.md" />
  </Folder>
  <Folder name=".vscode" defaultOpen>
    <File name="mcp.json" />
  </Folder>
  <File name="AGENTS.md" />
</Files>

**Key Transforms:**

- Identity passed through as `AGENTS.md`
- Global instructions merged into `.github/copilot-instructions.md`
- Skills exported as `.github/instructions/{name}.instructions.md`
- MCP capabilities mapped to `.vscode/mcp.json`
- Hooks converted to `.github/hooks/agentflow-hooks.json`

**Fidelity:** Medium. Some AgentFlow features require merging into fewer files.

**Import Support:** Yes. Can import existing Copilot configurations into AgentFlow.

</Tab>

## openclaw

Exports to OpenClaw's hook-based structure.

<Files>
  <Folder name="hooks" defaultOpen>
    <Folder name="hook-name" defaultOpen>
      <File name="HOOK.md" />
    </Folder>
  </Folder>
  <File name="AGENTS.md" />
  <File name="config.json" />
</Files>

**Key Transforms:**

- Identity passed through as `AGENTS.md`
- Skills and workflows mapped to `hooks/{name}/HOOK.md`
- MCP capabilities mapped to `config.json`

**Fidelity:** Medium. Hook-centric model requires flattening workflow graphs.

**Import Support:** Yes. Can import existing OpenClaw configurations into AgentFlow.
