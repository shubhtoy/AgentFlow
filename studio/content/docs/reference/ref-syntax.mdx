## Resource Reference

References a resource by category and name to include it in context.

<TypeTable
  type={{
    syntax: {
      type: "'{{category/name}}'",
      description: "Loads a resource into the node's context.",
    },
    categories: {
      type: "string",
      description: "instructions, capabilities, skills, memory, hooks",
    },
    resolution: {
      type: "string",
      description: "Looks up the resource in the .agentflow/ directory by category folder and filename.",
    },
    validation_errors: {
      type: "string",
      description: "broken_ref (not found), ambiguous_ref (multiple matches), unknown_category.",
    },
  }}
/>

```markdown
{{instructions/code-style}}
{{capabilities/write-file}}
{{skills/review-design}}
```

## Unconditional Edge

Connects two nodes with an always-taken path.

<TypeTable
  type={{
    syntax: {
      type: "'{{-> nodes/target}}'",
      description: "Defines a direct edge to the target node.",
    },
    resolution: {
      type: "string",
      description: "Resolves the target node by folder name within the same workflow.",
    },
    validation_errors: {
      type: "string",
      description: "unresolved_edge, self_referencing_edge, duplicate_edge.",
    },
  }}
/>

```markdown
{{-> nodes/create-design}}
```

## Conditional Edge

Connects two nodes with a condition-gated path. The condition is inline text after the pipe `|`, or a reference to an instruction file.

<TypeTable
  type={{
    syntax: {
      type: "'{{-> nodes/target | condition text}}' or '{{-> nodes/target | instructions/name}}'",
      description: "Defines an edge taken only when the condition is met.",
    },
    resolution: {
      type: "string",
      description: "Resolves the target node. Condition is either inline text or resolved from an instruction file.",
    },
    validation_errors: {
      type: "string",
      description: "unresolved_edge, missing_condition (when instruction ref doesn't exist).",
    },
  }}
/>

```markdown
<!-- Inline condition text -->
{{-> nodes/plan-tasks | design approved by user}}
{{-> nodes/create-design | design rejected or changes requested}}

<!-- Instruction reference as condition -->
{{-> nodes/plan-tasks | instructions/approval-criteria}}
```

When referencing an instruction, the instruction content (with narrativeTemplate wrapping) becomes the condition description.

## Data Flow

Passes output from a prior node into this node's context.

<TypeTable
  type={{
    syntax: {
      type: "'{{<< output.node-name}}'",
      description: "Loads the output of a preceding node into this node's context.",
    },
    resolution: {
      type: "string",
      description: "Resolves the source node and loads its stored output.",
    },
    validation_errors: {
      type: "string",
      description: "broken_data_flow (node not found), data_flow_forward (node hasn't executed yet).",
    },
  }}
/>

```markdown
Using the design from {{<< output.create-design}}, implement the feature.
```

Data flow refs can only point backward in the graph — to nodes that execute before the current one.

  }}


## Environment Variables

<TypeTable
  type={{
    syntax: {

    resolution: {
