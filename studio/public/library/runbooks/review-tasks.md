# Review Task List

Present the implementation task list to the user for review.

## What to present
- The ordered list of tasks with subtasks
- Dependency relationships between tasks
- Checkpoint tasks and their purpose
- Estimated scope per task
- Which requirements each task satisfies

## User options
- **Approve** — task list is correct, begin implementation
- **Reject** — reorder tasks, change granularity, add/remove tasks
- **Edit** — make direct changes to the task list
