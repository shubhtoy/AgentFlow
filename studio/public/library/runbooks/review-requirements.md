# Review Requirements

Present the requirements document to the user for review.

## What to present
- The full requirements document with numbered requirements and acceptance criteria
- A summary of scope: what's included and what's explicitly excluded
- Any assumptions made during elicitation
- Open questions that need user input

## User options
- **Approve** — requirements are complete and correct, proceed to design
- **Reject** — provide feedback on what needs to change (missing requirements, incorrect criteria, scope issues)
- **Edit** — make direct changes to the requirements document
