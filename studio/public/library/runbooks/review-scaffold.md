# Review Scaffold

Present the generated scaffold to the user for review before continuing with implementation.

## What to present
- File and directory structure created
- Boilerplate code and configuration files
- Key dependencies and their versions
- Any conventions or patterns applied

## User options
- **Approve** — scaffold looks good, proceed with implementation
- **Reject** — provide feedback on structure, naming, or configuration choices
- **Edit** — make direct changes to the scaffolded files
