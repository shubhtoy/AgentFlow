# Detailed Plan

Generate a comprehensive execution plan for a complex task, breaking it into phases with dependencies, risks, and verification steps.

## Process Steps
1. Understand the goal and constraints
2. Research the codebase and existing patterns
3. Decompose into phases and tasks
4. Identify dependencies between tasks
5. Add verification steps and rollback plans
6. Present the plan for approval before execution

## Plan Structure
- **Goal**: what we're trying to achieve
- **Scope**: what's in and what's out
- **Phases**: ordered groups of related tasks
- **Tasks**: specific actions with clear completion criteria
- **Dependencies**: which tasks block others
- **Risks**: what could go wrong and mitigation strategies
- **Verification**: how to confirm each phase succeeded

## Anti-Patterns
- Planning without reading the existing code first
- Creating tasks that are too large to verify independently
- Missing dependencies between tasks
- Skipping the verification step for each phase
