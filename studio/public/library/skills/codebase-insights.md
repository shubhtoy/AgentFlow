# Codebase Insights

Analyze a codebase to understand its structure, patterns, tech stack, and areas of concern.

## Process Steps
1. Scan directory structure and identify key directories
2. Read package manifests (package.json, requirements.txt, go.mod, etc.)
3. Identify frameworks, libraries, and patterns in use
4. Analyze code organization and architecture patterns
5. Find potential issues: large files, circular dependencies, dead code
6. Report findings with actionable observations

## What to Analyze
- **Tech stack**: languages, frameworks, build tools, test frameworks
- **Architecture**: monolith vs microservices, layer separation, module boundaries
- **Code health**: file sizes, complexity hotspots, test coverage gaps
- **Conventions**: naming patterns, directory structure, import style
- **Dependencies**: count, freshness, known vulnerabilities

## Output Format
A structured report with: tech stack summary, architecture overview, code health metrics, notable patterns, and areas of concern.
