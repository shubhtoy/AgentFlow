# Writing Style Guide

Ensure clear, professional, audience-appropriate writing.

## Process Steps
1. Identify the audience and calibrate tone (technical, executive, general)
2. Write the first draft — get ideas down without editing
3. Structure with headings, short paragraphs, and bullet points for scannability
4. Edit for clarity: active voice, concrete nouns, strong verbs, no filler
5. Cut 20% — remove anything that doesn't serve the reader
6. Read aloud — if it sounds awkward, rewrite it

## Clarity
- One idea per sentence
- Active voice over passive ("The system processes" not "is processed by")
- Concrete nouns and strong verbs
- Remove filler words (very, really, basically, actually, just)

## Structure
- Lead with the most important information
- Use headings to create scannable hierarchy
- Short paragraphs (3-4 sentences max)
- Bullet points for lists of 3+ items

## Tone Calibration
- **Technical audience**: precise terminology, skip basics, show code
- **Executive audience**: outcomes first, metrics, brief
- **General audience**: plain language, analogies, no jargon
- **Support audience**: empathetic, step-by-step, reassuring

## Grammar Checklist
- Subject-verb agreement
- Consistent tense throughout
- Proper comma usage (Oxford comma preferred)
- No dangling modifiers

## Review Process
1. Read aloud — if it sounds awkward, rewrite
2. Cut 20% — first drafts are always too long
3. Check every heading tells the reader what they'll learn
4. Verify all claims have sources

## Anti-Patterns
- Writing for yourself instead of the reader
- Using jargon without defining it for the target audience
- Wall-of-text paragraphs with no headings or structure
- Passive voice that hides who is responsible ("mistakes were made")
- Not proofreading — typos undermine credibility

## Output Format
A polished document with: clear heading hierarchy, short paragraphs, bullet points where appropriate, consistent tone matching the audience, and no filler words.

## Examples

### Example: Rewrite a vague paragraph
**Before**: "The system is basically used by various users to do different things with their data and it's really important that it works properly."

**After**: "Three user roles — admin, editor, and viewer — interact with the data pipeline. Admins configure sources, editors transform records, and viewers access dashboards."
