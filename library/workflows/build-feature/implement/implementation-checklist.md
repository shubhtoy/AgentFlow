# Implementation Checklist

For each task, complete every step before moving to the next task:

- [ ] Read the task summary and acceptance criteria
- [ ] Write a failing test that captures the acceptance criteria
- [ ] Implement the minimum code to make the test pass
- [ ] Apply error handling at boundaries
- [ ] Refactor with tests green — simplify, extract, rename
- [ ] Run the full test suite — no regressions
- [ ] Self-review the diff — check for security, clarity, coverage
- [ ] Commit with a descriptive message
- [ ] Update session notes with task status and any decisions made

If any step fails, stop and resolve before continuing.
