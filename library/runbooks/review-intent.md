# Review Intent

Present the proposed intent to the user for review before proceeding with execution.

## What to present
- Summary of the interpreted user request
- Planned approach and key steps
- Assumptions made during intent analysis
- Scope boundaries — what will and will not be addressed

## User options
- **Approve** — intent is correct, proceed with execution
- **Reject** — provide feedback to refine the intent or request a different approach
- **Edit** — make direct changes to the intent description
