# Session Notes

Working notes and temporary context from the current session. Scratch space for observations that may be promoted to permanent memory.

## Format

```
[timestamp] Note: <observation or finding>
Confidence: low | medium | high
Promote to: <project-context | user-preferences | lessons-learned | team-context | discard>
```

## Lifecycle
Session notes are ephemeral. At session end, review and:
1. Promote confirmed patterns to project-context
2. Promote confirmed preferences to user-preferences
3. Promote confirmed lessons to lessons-learned
4. Discard notes that are no longer relevant

## When to Read
- To recall what was discovered earlier in this session
- Before session-end memory review

## When to Write
- When you notice something that might be a pattern but aren't sure yet
- When the user says something that sounds like a preference but hasn't been confirmed
- Freely during the session — cleanup happens at review time
