# Team Context

Org-wide knowledge that applies across multiple repositories and projects.

## Format

```
[YYYY-MM-DD] Category: <infrastructure | convention | ownership | tooling | process>
Fact: <concise statement>
Scope: <org-wide | team-name>
```

## When to Read
- When starting work on any repo — understand org context
- Before making infrastructure or cross-cutting decisions
- When the user references team processes or shared services

## When to Write
- When the user shares org-wide knowledge
- When you discover cross-repo patterns
- Verify with the user before writing — org facts affect everyone
