# User Preferences

Personal instructions and preferences specific to this user. Follows you across projects.

## Format

```
[YYYY-MM-DD] Category: <style | workflow | communication | tooling>
Preference: <concise statement>
```

## When to Read
- Start of every session — tailor behavior to the user
- Before generating code — match their style
- Before explaining something — match their preferred communication level

## When to Write
- When the user explicitly states a preference
- When the user corrects you — that correction IS a preference
- Update existing entries if a preference changes
- Don't infer from a single instance — wait for a clear pattern or explicit statement
