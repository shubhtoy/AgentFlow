# Lessons Learned

Mistakes, failed approaches, and the correct solutions discovered afterward.

## Format

```
[YYYY-MM-DD] Problem: <what went wrong>
Wrong approach: <what was tried and failed>
Correct approach: <what actually fixed it>
Impact: low | medium | high
```

## When to Read
- When encountering an error — a past lesson may apply directly
- Before attempting a complex refactor or migration
- When a user says "we tried that before" or "that won't work"

## When to Write
- After resolving a non-obvious bug that took significant effort
- When the user says "remember this" or "don't do that again"
- When an approach fails and a different solution is found
- When a user corrects you — the correction encodes a lesson
