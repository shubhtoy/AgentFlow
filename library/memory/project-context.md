# Project Context

Project-specific facts, conventions, and instructions that all contributors and agents should follow.

## Format

```
[YYYY-MM-DD] Category: <stack | convention | architecture | deployment | domain>
Fact: <concise statement>
```

## When to Read
- Start of every session — this is ground truth for the project
- Before generating code — match established patterns
- Before proposing architecture or tooling changes

## When to Write
- When the user shares a project fact ("we use PostgreSQL", "deploys go through CI")
- When you discover a non-obvious codebase pattern
- Update existing entries when facts change
- Only record stable, long-lived facts — not transient information
