# AI Chat Integration — Architecture & Context

## Overview

AgentFlow's copilot sidebar is powered by CopilotKit v2 + LangGraph.js. The agent runs as a separate LangGraph server process, connected to the Next.js frontend via CopilotKit's `LangGraphAgent` adapter.

## Architecture

```
┌─────────────────────────────────────────────────┐
│  Browser (Next.js)                              │

│       │                    │                    │
│  ModelPicker (SWR)    CopilotActions (v2)        │
│  SettingsDialog       CopilotReadables (v2)      │
│  ClearChat            CopilotToolRenderers (v2)  │
│       │                    │                    │
│       ▼                    ▼                    │
│  POST /api/copilotkit/...                       │
└────────────────┬────────────────────────────────┘
                 │
                 ▼
┌─────────────────────────────────────────────────┐
│  Next.js API Route (route.ts)                   │
│                                                 │
│  CopilotRuntime + LangGraphAgent                │
│       │                                         │
│       ▼ HTTP                                    │
│  localhost:2024 (LangGraph dev server)           │
└────────────────┬────────────────────────────────┘
                 │
                 ▼
┌─────────────────────────────────────────────────┐
│  LangGraph Agent (agent.ts)                     │
│                                                 │
│  StateGraph: agent → tools → agent loop         │
│       │                                         │
│  resolveModel() → createChatModel()             │
│       │                                         │

└─────────────────────────────────────────────────┘
```

## Key Files

### Backend

| File | Purpose |
|------|---------|
| `lib/copilot/agent.ts` | LangGraph ReAct agent graph. Loads dotenv, resolves model per-request, binds server + frontend tools. |
| `lib/copilot/model-registry.ts` | Model catalog (from `models.json`), filtering, selection persistence, provider routing via `initChatModel`. |
| `lib/copilot/key-store.ts` | API key management. Default mode: `.env.local`. Multi-user (noEnv): per-session JSON file with 24h TTL auto-cleanup. |
| `lib/copilot/system-prompt.ts` | Comprehensive system prompt with AgentFlow domain knowledge, tool guidance, security rules. |
| `lib/copilot/server-tools.ts` | `defineTool` server tools for CopilotKit BuiltInAgent (legacy, kept for reference). |
| `lib/copilot/models.json` | Static model catalog generated by `npm run fetch-models` from OpenRouter API. |
| `app/api/copilotkit/[[...path]]/route.ts` | CopilotKit runtime endpoint. Connects to LangGraph server via `LangGraphAgent`. |
| `app/api/copilot/model/route.ts` | Model list + selection API. Merges static catalog with Groq runtime discovery. Supports query filters. |
| `app/api/copilot/keys/route.ts` | API key CRUD. Default mode writes `.env.local`, multi-user mode writes `.copilot-keys.json`. |
| `langgraph.json` | LangGraph CLI config pointing at `agent.ts:graph`. |

### Frontend

| File | Purpose |
|------|---------|
| `components/copilot/CopilotPanel.tsx` | Sidebar layout with CopilotSidebar, model picker, context indicator, clear chat button, tools menu. |
| `components/copilot/ModelPicker.tsx` | Model selection dropdown. SWR for data fetching. Filter chips (free/tools/fast/vision/reasoning/code). Shows context length, FREE badge, grouped by provider. |
| `components/copilot/CopilotActions.tsx` | 15 frontend tools via `useFrontendTool` (v2, Zod schemas). File ops, validation, UI control, MCP management. |
| `components/copilot/CopilotReadables.tsx` | 5 context items via `useAgentContext` (v2). Workspace graph, selection, UI state, validation, MCP config. |
| `components/copilot/CopilotToolRenderers.tsx` | Tool call cards in chat via `useRenderTool` (v2). |
| `components/copilot/CopilotSuggestions.tsx` | Dynamic suggestion pills via `useConfigureSuggestions` (v2). |
| `components/copilot/CopilotProvider.tsx` | Root `CopilotKitProvider` with error handling. |
| `components/SettingsDialog.tsx` | API key configuration UI. Shows all 8 provider keys with status, masked previews, direct links. OpenRouter callout for free models. |

## Model Registry

### Data Source

`npm run fetch-models` fetches from OpenRouter's public API (`/api/v1/models`), filters by:
- Major providers only (OpenAI, Anthropic, Google, Meta, Mistral, DeepSeek, Qwen, xAI, NVIDIA, Nous, etc.)
- Text input + text output
- Not expired
- Context >= 4096 tokens

Writes `lib/copilot/models.json` (~249 models, ~24 free).

### Groq Models

Fetched at runtime from `api.groq.com/openai/v1/models` when `GROQ_API_KEY` is available. Cached 10 min. Merged into the model API response. Filters out non-chat models (whisper, guard, tts, embed).

### Provider Routing

| Model ID pattern | Route | Key needed |
|---|---|---|
| `openai/gpt-4o` | `@langchain/openai` | `OPENAI_API_KEY` |
| `anthropic/claude-sonnet-4` | `@langchain/anthropic` | `ANTHROPIC_API_KEY` |
| `google/gemini-2.5-flash` | `@langchain/google-genai` | `GOOGLE_API_KEY` |
| `groq/llama-3.3-70b-versatile` | `@langchain/groq` | `GROQ_API_KEY` |
| `deepseek/deepseek-chat` | `@langchain/openai` + baseURL | `DEEPSEEK_API_KEY` |
| `x-ai/grok-3` | `@langchain/openai` + baseURL | `XAI_API_KEY` |
| `mistralai/mistral-large` | `@langchain/openai` + baseURL | `MISTRAL_API_KEY` |
| `meta-llama/llama-3.3-70b-instruct:free` | OpenRouter API | `OPENROUTER_API_KEY` |
| Any `:free` model | OpenRouter API | `OPENROUTER_API_KEY` |

### Free Models

All `:free` suffix models route through OpenRouter's OpenAI-compatible API. Only needs `OPENROUTER_API_KEY` (free, no credit card). ~24 free models from Google, Meta, Qwen, Mistral, NVIDIA, Nous, etc.

## Key Management

### Default Mode (single user)
- Keys in `.env.local`
- `POST /api/copilot/keys` writes to `.env.local` + hot-patches `process.env`
- LangGraph agent loads `.env.local` via dotenv at startup

### Multi-User Mode (noEnv)
- Keys per session in `.copilot-keys.json`
- Each entry has a 24h TTL, stale sessions auto-pruned on every read/write
- Both Next.js and LangGraph processes read the same file
- Session ID from `af-session` cookie

## Running

```bash
# Start both servers
npm run dev:all

# Or separately:
npm run agent    # LangGraph server on :2024
npm run dev      # Next.js on :3000

# Refresh model catalog from OpenRouter
npm run fetch-models

# Kill stuck LangGraph processes
npm run agent:kill
```

## CSS Theme Bridge

CopilotKit styles scoped under `[data-copilotkit]`. AgentFlow's HSL theme vars bridged via `--af-*` aliases in `globals.css`. Font sizes, spacing, prose overrides all in the `[data-copilotkit]` block.

## Known Limitations

- LangGraph CLI spawns worker threads that don't exit cleanly — use `npm run agent:kill` if stuck
- Token usage tracking requires CopilotKit Premium (`publicLicenseKey`)
- Chat history doesn't persist across page refreshes (MemorySaver is in-memory)
- Sub-agents and workflow execution are future work
- BuiltInAgent only supports openai/anthropic/google natively — other providers need LangGraph path
